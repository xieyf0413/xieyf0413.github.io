arcgis安装与地图发布

发表于 2016-04-19   |   分类于 地图   |  

工作中经常用到地图服务，最早用的google map，后来由于众所周知的原因没法用了，后来改用了arcgis；每次有一个新项目就需要发布一个新的地图，一般都是我来发布，其他同事也很少去做这个事情，碰到自己忙的时候就会耽误事了。这几天趁着有点时间，打算把arcgis的安装过程和地图服务的发布过程写下来，这样大家如果想自己试试，可以参考学习。

Arcgis的介质均为10.0版本，如有需要，可以找我索取，Email:xieyf0413@gmail.com。

修改动态地图服务为瓦片地图服务

在arcgis的服务中，动态地图服务和瓦片地图服务是最常见的两种地图服务，他们的区别就在于，在使用动态地图服务时，客户端会一次性将整张地图全部拿到后，进行呈现和渲染；而在使用瓦片地图服务时，服务端会将整张地图进行切片，根据比例尺的不同将地图切成一张一张的小图片，客户端使用时只会加载当前可视范围内的小图片；我们的应用程序通常使用瓦片地图服务比较多。

通过上边步骤发布出来的地图服务是动态地图服务，所以，我们需要将其修改为瓦片地图服务，方法如下：

一.从系统的 开始菜单，找到ArcGis\ArcCatalog，双击”Add ArcGIS Server”，选择”Manage GIS Services”,然后下一步：

二.输入服务的url和地址，点击完成：

三.可以看到左侧的树上已经能够看到服务的ip和已经发布的地图服务：

在”hebeimap”的右健中点击“Service Properties…”菜单，弹出如下窗口，选择”Caching”的标签

四.修改红色框中的几个地方：

  • 选择”Using tiles from the cache that you will define below”,使用瓦片地图服务；
  • Scales设置为12；（这里设置的是地图的缩放级别，可以根据实际情况设置）
  • 选择”Create tiles on demand”，此处非常重要，一定要选中；选中的意思是这个切片的过程在使用时才切，如果不选那么就会立即开始切，切的过程会非常非常慢。

好了，咱们的服务类型也修改好了。

把地图发布为REST服务

通过上边的方式发布的地图为SOAP服务，但是我们在使用时，比如通过javascript、flex在使用时却是要使用REST服务的，所以需要重新发布一次，过程如下：

一.打开arcgis server manager页面，登录进去之后，选择左侧导航树中的 services-> Configure Service Handler菜单 :

二.选择”export”选项卡：

  • Export Web Services ，选择”rest service”；
  • War file name，可以随便写个比如：rest；
  • Soap url：http://ip:8399/arcgis/services ；
  • User name/Password：登录arcigsmanager的用户名和密码；

点击”Test Connection”，测试成功；

点击最下方的导出，生成rest.war文件，保存在磁盘上。

三.把上一步导出的rest.war拷贝到%your ArcGIS Instal Directory%\Server10.0\java\manager\web_output\web目录下；

四.在服务中，重启arcgis相关的几个服务；

五.重启完等待一小会儿，%your ArcGIS Instal Directory%\Server10.0\java\manager\web_output\web目录下应该生成了rest的一个子目录；

六.打开rest服务的地址，http://ip:8399/rest/services，可以看到已经发布的rest服务，大功告成。
